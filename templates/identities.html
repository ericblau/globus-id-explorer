Apps can use Globus's Identities API to retrieve information about an identity using either a username (username@identity.provider) or a sub (aka id). The Identities API returns the following information about the sub in your ID Token. Explanations follow the data below. To learn how to correct this data, click here.

{{globusmyids}}

In the identities section, the name, organization, email, and identity_provider are the same as in your ID Token. The id is the same as the ID Token sub. The username is the same as the ID Token preferred_username.

Some identity providers—especially academic institutions and schools—report anonymized usernames to prevent misuse of your username. In the username field, you might see a random string of numbers and letters instead of your actual username. The same string will be used consistently every time you login.

Applications may also request information about the identity provider for an identity. This information is provided in the included section. ({{pagetitle}} requested this information.) For the identity provider, the id is a unique identifier that will never be used for a different identity provider. The name and short_name are human-readable names for the identity provider. The alternative_names list other human-readable names for the same identity provider. The domains list unique, machine-readable domains, used in identity usernames and preferred_usernames, for which this identity provider is authoritative. This identity provider is the only one that may issue identities in these domains.
